
对现有海报墙结构以及渲染逻辑进行完全的重构 删除掉原有的旧的不需要的代码

## 设计方案

### 1. 扁平化数据结构

将当前的分层结构（分组标题+电视剧卡片）扁平化为一个统一的列表，每个元素包含类型信息和位置信息：

```javascript
[
  {type: "header", group: "A", x: 0},
  {type: "item", tvShow: {...}, x: 160, n: 0},
  {type: "item", tvShow: {...}, x: 160, n: 1}, 
  {type: "item", tvShow: {...}, x: 160, n: 2},
  {type: "header", group: "B", x: 470},
  {type: "item", tvShow: {...}, x: 630, n: 0},
  {type: "item", tvShow: {...}, x: 630, n: 1}
]
```

### 2. 位置计算逻辑

根据当前的排序类型和分组策略，预先计算每个元素的位置：
1. **横向位置（扁平数据中x）计算**：
   - 每列可容纳的海报卡片数量：`maxPossibleRows`
   - 每个标题占完整一列
   - 每增加一个标题列，x增加`posterWidth/2 + gap`
   - 每增加一个海报卡片列，x增加`posterWidth + gap`

2. **纵向位置计算**：
   - 标题元素的y位置始终为0
   - 海报卡片的y位置为`n * (posterHeight + gap)`

### 3. 缓冲区域管理

定义缓冲区域来决定哪些元素需要被渲染：

distance_x、maxPossibleRows、posterWidth、posterHeight、gap等均使用源代码中的计算方式

- **缓冲区域左侧**：`distance_x + (bodyWidth/4)`
- **缓冲区域右侧**：`distance_x + (bodyWidth/4)*3`
- **渲染条件**：元素的x位置在缓冲区域内时创建DOM，超出区域时销毁DOM

### 4. 网格构建逻辑

使用手动计算位置的方式构建网格，不依赖CSS Grid：

1. **元素定位**：
   - 每个元素的x位置为`x - distance_x`（随滚动距离变化）
   - 每个元素的y位置为`n * (posterHeight + gap)`（标题为0）

2. **DOM管理**：
   - 进入缓冲区域的元素创建DOM并添加到容器
   - 离开缓冲区域的元素从容器移除并销毁DOM
